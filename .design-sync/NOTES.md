# design-sync NOTES — planner

This repo is a **Next.js application**, not a published component package. The
design system is the 44 shadcn/ui primitives in `components/ui/`. There is no
`dist/` and no Storybook, so the converter runs in **synth-entry (package) mode**.

## How the build runs (reproducible)

1. `node .design-sync/compile-css.mjs` — set as `cfg.buildCmd`. Compiles Tailwind v4
   (`app/globals.css`) to `ds-tailwind.css` (gitignored) and prepends a Google-Fonts
   `@import` + `--font-jakarta/--font-manrope/--font-geist-mono` bindings. `cfg.cssEntry`
   points at `ds-tailwind.css`. WITHOUT this the styles closure is raw `@import "tailwindcss"`
   (no utilities) → every preview renders unstyled.
2. `node .ds-sync/package-build.mjs --config .design-sync/config.json --node-modules ./node_modules --entry ./components/ui/__ds_synth__.ts --out ./ds-bundle`
   - The `--entry` path is **intentionally non-existent**: it forces `PKG_DIR` to resolve to
     the repo root (walks up to the `planner` package.json) AND makes `resolveDistEntry` return
     null → synth-from-src discovery over `cfg.srcDir = components/ui`.
3. `node .ds-sync/package-validate.mjs ./ds-bundle`

## Component scoping

- Synth discovery finds **229** PascalCase exports (all shadcn sub-parts). `cfg.componentSrcMap`
  nulls the **185** sub-parts so only the **44 top-level** components get cards. The bundle still
  exports all 229 (synth entry = `export *` from every file), so the design agent can still
  compose `DialogContent`, `SelectItem`, etc. — they just don't each get a card.
- IMPORTANT: synth-mode auto-discovery (`deriveComponentsFromSrc`) only runs when the component
  list is empty. Do NOT add non-null `componentSrcMap` pins — a single pin short-circuits
  discovery and you get only the pinned components. (Discovery already honors the `null` excludes.)

## Forks / overrides

- `.design-sync/overrides/story-imports.mjs` (`cfg.libOverrides`) — **perf fork.** Skips DS
  import policy for imports whose importer is inside `node_modules`. Without it, each preview
  esbuild-bundled `lucide-react`'s ~1600-icon barrel through the catch-all async resolver →
  ~90s/preview → ~70min full builds. With it, plain esbuild resolves third-party internals
  natively (~0.4s) → full build is a few minutes. Safe: DS shim/source-barrel policy only ever
  applies to the story file's own imports + the package's own source, never to third-party
  internals.
- `.ds-sync/package-capture.mjs` `settle()` — **recharts patch (NOT committed; staged script is
  gitignored & regenerated on re-sync — see Re-sync risks).** Adds a bounded wait for
  `.recharts-surface` to acquire non-zero width before screenshot. recharts 3.8 ResponsiveContainer
  paints on a later ResizeObserver frame, so ChartContainer screenshots blank without it.

## Overlay / wide card overrides (`cfg.overrides`)

- viewport+single (render open in-card): Dialog, AlertDialog, Sheet, ResponsiveDialog, Popover,
  HoverCard, Tooltip, DropdownMenu, ContextMenu, Combobox, Command, Select, Sidebar.
- column (full-width rows): Table, ChartContainer, ResizablePanelGroup.
- Adding/changing a `viewport` override re-keys that component's grade slice → `preview-rebuild`
  fails `[CONFIG_STALE]` until a full `package-build` re-stamps `.stories-map.json`. So: set all
  overrides BEFORE the build whose stamp the subagents will rely on. (`cardMode:column` alone
  does NOT re-key; `viewport` does.)

## Known render warns (triaged legitimate)

- Skeleton renders as a muted shimmer block with no text — correct, not blank.
- Toaster (`sonner`) renders toasts imperatively; a bare `<Toaster/>` is empty. The preview
  shows a representative toast surface mirroring sonner's `--normal-bg/--normal-border` tokens.
- ContextMenu (Radix) has no `open`/`defaultOpen` — only opens on right-click. Static capture
  shows the trigger; DropdownMenu covers the same menu visual.

## Preview authoring gotchas

- Previews import DS components from the bare specifier `"planner"` (→ `window.PlannerUI`),
  icons from `"lucide-react"`, `cn` from `@/lib/utils`.
- Tailwind v4 tree-shake: a utility class only compiles into `ds-tailwind.css` if it appears in
  scanned source. `compile-css.mjs` runs the CLI from repo root so it scans `components/` AND
  `.design-sync/previews/` — but a brand-new arbitrary class (`w-[460px]`) used only in a preview
  needs a CSS recompile after authoring. Prefer common utilities or inline `style`.

## Re-sync risks (watch-list for the next run)

- **`.ds-sync/package-capture.mjs` recharts `settle()` patch is NOT durable** — `.ds-sync/` is
  gitignored and re-copied from the skill on every re-sync. After re-staging scripts, RE-APPLY
  the recharts wait in `settle()` (bounded poll for `.recharts-surface` width), or ChartContainer
  will re-grade blank. Consider upstreaming.
- `ds-tailwind.css` is regenerated by `cfg.buildCmd`; the Google-Fonts `@import` loads at runtime
  (`[FONT_REMOTE]`, expected) — fonts are NOT shipped in `fonts/`.
- The non-existent `--entry` trick depends on the repo root `package.json` being named `planner`.
- If `components/ui/` gains/removes files, re-check the `componentSrcMap` null set (sub-parts of a
  new compound component will appear as new top-level cards otherwise).
