## Anti-Patterns Verdict

LLM assessment: This does NOT read as AI slop. It passes the product test (a Linear/Notion/Stripe-fluent user would trust it): familiar patterns used correctly, a coherent OKLCH token system, real interaction states, DST-correct recurrence, reduced-motion + safe-area handling, contrast annotated in the tokens, and full Catppuccin theming. It is visibly hand-crafted. The few tells: the `border-l-4` side-stripe on task cards (a known AI tell, though here it defensibly carries the task color), the cool rainbow event palette fighting the warm brand, and slightly generic auth/empty surfaces.

Deterministic scan: 2 findings, both the side-tab stripe (`task-card.tsx:74`, `task-backlog-rail.tsx:74`). No gradient text, no glassmorphism, no eyebrow/numbered-section scaffolding, no hero-metric template. Clean for an app this size.

Overlay: Extensive live browser inspection (17 screenshots, light+dark, desktop+mobile) but the detect.js overlay was not injected, so no user-visible overlay exists.

## Overall Impression

This is a genuinely well-built product calendar, not a prototype. The token system, theming, recurrence/DST correctness, sharing model, and task subsystem are all polished. The biggest opportunities are not "make it look designed", they are: (1) fix real mobile breakage, (2) tame the visual heaviness of the core week view under real data, and (3) make the considerable hidden power discoverable.

## What's Working

1. Event + task editing. Large title field, progressive disclosure ("More options"), a Sharing segmented control with inline plain-language explanation, and a full subtask panel (progress bar, "Do in order", blocked lock badge, "Add to calendar"). Product-grade.
2. The design system. Warm terracotta + teal on stone, soft warm-tinted shadows, accessible contrast baked into token comments, OKLCH color-mix toasts, and a complete accent/tone/Catppuccin theming layer. Dark mode is coherent and intentional.
3. Real-data resilience. Dense, overlapping, multilingual (Cyrillic/French) events render with working overlap packing, graceful truncation, and de-emphasized inactive (sleep) events.

## Priority Issues

- [P1] Mobile Tasks toolbar overflows the viewport. Measured 94px of horizontal overflow at 390px (`scrollWidth` 469 vs 390); the right cluster clips the primary "New task" button and forces a horizontal scroll. Cause: `tasks-toolbar.tsx` packs board switcher + nav + view toggle + actions into one non-collapsing row, unlike the calendar toolbar which collapses into a `⋯` menu on mobile. Fix: apply the same mobile compaction to the tasks toolbar. Command: /impeccable adapt.
- [P1] Week view is unusable on phones. Seven columns at 390px shrink every event to a single letter ("Z.", "W…", "S.") with truncated times. The app smartly defaults mobile to Agenda and offers Day/3-Day, but Week stays selectable and is near-illegible when chosen. Fix: drop Week from the mobile view switcher (or make it horizontally scrollable with a min column width), defaulting phones to Day/3-Day/Agenda. Command: /impeccable adapt.
- [P2] Overnight "Sleep"/inactive blocks dominate week + day. The 21:00–07:00 sleep events render as tall dark slabs filling 00:00–07:00 of every column, heavy and gloomy, competing with real events. The "show inactive in month" toggle does not extend to week/day. Fix: render inactive overnight events as a thin tinted gutter/hatched band rather than full-height blocks, and let the toggle apply to week/day. Command: /impeccable layout.
- [P2] Event color palette fights the brand. Category colors default to a cool rainbow (blue/purple/teal/pink/green) that clashes with the warm terracotta shell, reading as incoherent. Fix: ship a warm-leaning default category palette (or harmonize hues / lower per-event saturation), reserving a cool family for the partner's layer. Command: /impeccable colorize.
- [P2] Hidden power, no discoverability or onboarding. Shortcuts (mask titles, sidebar toggles, undo), drag-resize, and the secondary timezone have no in-app surfacing, and there is no first-run guidance for an empty calendar. Fix: add a "?" shortcuts sheet plus subtle hints, and a light empty-calendar onboarding. Command: /impeccable onboard.

## Persona Red Flags

Alex (Power User): keyboard shortcuts exist but are undiscoverable (no cheat sheet); no visible command palette despite `cmdk` being installed; no bulk task actions. Otherwise fast.

Sam (Accessibility): strong baseline — focus rings, aria-labels on icon buttons, reduced-motion respected, zoom never disabled, contrast tuned. Flags: muted body text sits right at the 4.5:1 floor with no margin; overdue is signaled mainly by red text; mobile week's single-letter blocks fail low-vision users badly.

Casey (Distracted Mobile): agenda default is excellent, but the primary create "+" is top-right (outside the thumb zone), the Tasks toolbar overflow clips "New task" (P1), and Week is unreadable (P1).

## Minor Observations

- Pre-auth 401 storm: data queries fire on /login before a session exists (console noise, wasted requests).
- "Contexts" is unusual wording for what users elsewhere call calendars/categories.
- Bare hour labels ("01".."23") on the time axis are slightly cryptic.
- Settings is a long single-column scroll on desktop with large empty side margins; an anchored sub-nav or two-column layout would help on wide screens.
- Destructive Delete is styled near-primary; it relies on undo rather than a confirm (a defensible modern pattern, worth a deliberate decision).
- The event-details popover has a small unlabeled icon button between Share and Edit.
- Agenda rows look stretched/sparse on wide desktop (full-width rows, large title-to-time gap).

## Questions to Consider

- Should Week even be offered on phones, or is Day/3-Day/Agenda enough?
- Should overnight/inactive events become a first-class background layer (a tinted gutter) instead of full blocks?
- Could the default category palette lean warm to match the brand, with cool reserved for the partner's layer?
- For two people who know the app, what would make it feel like theirs (personality on login/empty states) rather than a generic calendar?
