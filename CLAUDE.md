# Design Context

This project uses the **impeccable** design skill (`/impeccable …`). Two root files
carry the design system — read them before any UI work:

- **PRODUCT.md** — register, users, product purpose, brand personality, anti-references,
  and the 5 strategic design principles.
- **DESIGN.md** — the visual system (color, typography, elevation, components) in the
  Stitch DESIGN.md format. `.impeccable/design.json` is its machine-readable sidecar.

**Register:** product (an app-shell planner, not a marketing surface).
**Personality:** calm, precise, understated — *"a warm, shared calendar for two."*
Warmth is carried by restraint (warm-paper surface, one warm-stone accent, with terracotta
surviving as Member A's identity), never by decoration.
**Avoid:** gamified-productivity patterns, generic-SaaS-dashboard chrome, and
cold/sterile minimalism.
